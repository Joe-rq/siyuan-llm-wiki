# LLM Wiki 工作流规范

## 目录结构

```
LLM-Wiki/
├── index.md              # 知识库入口，实体索引
├── log.md                # 变更日志
├── schema.md             # 本文档
├── raw/                  # 原始文档存档
└── wiki/
    ├── entities/         # 实体页（人物、概念、项目...）
    ├── topics/           # 主题页（跨实体综合）
    ├── sources/          # 来源摘要
    └── synthesis/        # 综合判断
```

## 链接格式

```
✅ 正确：((<docId> '显示标题'))
❌ 错误：[[标题]]           ← Obsidian 格式，不会写入 refs 表
❌ 错误：((<docId> 标题))   ← 标题缺少单引号
❌ 错误：<docId>            ← 裸 docId（残留文本块）
```

## 命令选择规则

| 文档类型 | 更新方式 | 命令 | 原因 |
|---------|---------|------|------|
| index | 重写整页 | `update` | 需要合并到已有分类，禁止 prepend |
| log | 重写整页 | `update` | 保持格式统一，避免裸 docId |
| 实体页 | 追加块 | `bi` | 块级追加新描述 |
| 主题页 | 追加块 | `bi` | 追加相关实体链接 |

## 实体标准分类

1. 笔记方法论
2. 人物
3. LLM/Agent 技术
4. 向量数据库
5. 技术概念
6. 投资/金融
7. 工具
8. 系统/概念

新增分类规则：新领域内容 ≥5 个实体且不匹配任何已有分类时，才可新增。
